# 鋼珠瑕疵檢測 — 實作規格

> **方案**：U-Net + EfficientNet encoder（透過 `segmentation_models_pytorch` / smp）做 semantic segmentation。
> 每像素分類成 13 類（12 瑕疵 + 背景），同時得到「是什麼瑕疵」+「在哪裡」。
> 另一個 YOLO 專案已存在 → 本專案走 detection-free 路線，做 detection vs segmentation 的方法論對照。

## 1. 任務定義

**目標**：對單顆鋼珠影像進行 **semantic segmentation**，每像素輸出類別。

**類別**（13 類）：
- Class 0：背景（鋼珠正常表面 / 影像背景）
- Class 1-12：12 種瑕疵

**輸入**：單顆鋼珠 RGB 影像（1201×1201）
**輸出**：H×W 的類別 map（每像素一個 class id）+ 對應 softmax 信心圖

**Good 樣本**：整張圖全部標為背景（class 0）。

## 3. 技術選型

| 元件 | 選擇 | 理由 |
|---|---|---|
| Framework | PyTorch + **segmentation_models_pytorch (smp)** | 一行建模、encoder/decoder 任意組合 |
| Decoder | **U-Net** | 邊界精細、社群成熟、適合工業瑕疵 |
| Encoder | **efficientnet-b0**（主）/ b3（備案） | ImageNet 預訓練、跟原 EfficientNet 研究脈絡延續 |
| Input size | **512×512**（從 1201 縮放） | 平衡解析度與訓練速度 |
| Loss | **CrossEntropy + Dice 組合** 或 **Focal + Dice** | 對抗 pixel 級不平衡 |
| Optimizer | AdamW，lr=3e-4，weight_decay=1e-4 | 標準 |
| Scheduler | CosineAnnealingLR，warmup 5 epochs | — |
| Epochs | 100（早停 patience=20） | — |
| Batch size | 8-16（依 GPU 記憶體） | 512×512 比較吃記憶體 |
| Augmentation | Albumentations（mask-aware） | smp 標配組合 |

### 為什麼選 smp + U-Net + EfficientNet

1. **跟原本 EfficientNet 的研究脈絡延續**——換 head 不換 backbone
2. **U-Net 邊界精細**——適合後續做瑕疵尺寸量測
3. **smp 套件成熟**——一行建模，encoder 可隨意換比較
4. **跟 YOLO 專案形成對照**——detection vs segmentation，方法論完整

## 4. 資料處理流程

### 4.1 LabelMe → Mask PNG 轉換（必做）

每張 .jpg 對應產生一張 mask.png：
- 大小同原圖 1201×1201
- 每像素值 = class id（0 = 背景，1-12 = 瑕疵類）
- 用 PIL.ImageDraw / cv2.fillPoly 把 polygon 內填上 class id

`convert_labelme_to_mask.py` 流程：
```python
for json_file in 所有 json:
    讀 imageHeight, imageWidth
    建立 H×W 全 0 mask
    for shape in shapes:
        class_id = 原 label "100"~"111" → 1~12 mapping
        cv2.fillPoly(mask, [points], class_id)
    儲存 mask 為 PNG（uint8）

# good 資料夾：直接產出全 0 mask
```

**驗證**：抽 10 張視覺化，疊圖確認 polygon 對齊。

### 4.2 Train/Val/Test 切分

> ⚠️ **[更改]** 原規格要求按日期批次切分，但 M1 盤點發現：
> - good 69 張全集中在 0413 批次，無法按批次分散到各 set
> - 1029 批次佔 60%，若整批進 train 則 val/test 缺少多數類別
> - 按批次切導致部分類別在 train/val/test 其中一個 set 完全缺席
>
> **改為 Stratified Random Split（60/20/20，seed=42）**：
> - 按類別做分層抽樣，確保每個 set 都有所有 13 類別
> - 同類別的圖片按比例分配到 train/val/test
> - 犧牲跨批次泛化的嚴謹性，換取每個 set 類別完整性

- 比例：60% train / 20% val / 20% test
- seed=42 固定可重現
- 確保每類別在三個 set 都有樣本

### 4.3 Augmentation（Albumentations，mask-aware）

```python
import albumentations as A

train_tf = A.Compose([
    A.Resize(512, 512),
    A.HorizontalFlip(p=0.5),
    A.VerticalFlip(p=0.5),
    A.Rotate(limit=30, p=0.5),
    A.RandomResizedCrop(512, 512, scale=(0.8, 1.0), p=0.3),
    # 對抗 batch effect 的關鍵
    A.ColorJitter(brightness=0.4, contrast=0.4, saturation=0.2, p=0.7),
    A.RandomGamma(gamma_limit=(70, 130), p=0.5),
    A.CLAHE(p=0.2),  # 局部對比度增強
    # 小幅模糊/噪聲提升泛化
    A.OneOf([A.GaussianBlur(p=1), A.MotionBlur(p=1)], p=0.2),
    A.Normalize(...),
    ToTensorV2(),
])

val_tf = A.Compose([
    A.Resize(512, 512),
    A.Normalize(...),
    ToTensorV2(),
])
```

Albumentations 會**同時 transform image 和 mask**（這是 mask-aware）。

### 4.4 長尾處理（pixel-level）

兩種互補做法：

**(A) Loss 層**：CrossEntropy 加 class weight + Dice loss
```python
# class weight ≈ 1 / sqrt(pixel_count_per_class)
ce_loss = nn.CrossEntropyLoss(weight=class_weights, ignore_index=255)
dice_loss = smp.losses.DiceLoss(mode='multiclass')
loss = ce_loss + dice_loss
```

**(B) Sampling 層**：含小類瑕疵的圖加大抽樣機率
- 計算每張圖含哪些 class
- 用 WeightedRandomSampler 讓小類圖被抽中機率提高

**(C) 備案：合併語意相近類**
- `小黑傷 + 大黑傷` 合併、`亮傷-暗 + 亮傷-亮` 合併
- 跟業務確認可不可以

## 5. 評估指標

**主指標**：
- **mean IoU（mIoU）**：每類 IoU 平均，標準 segmentation 指標
- **Per-class IoU**：找出哪類爛掉
- **Per-class Dice**：類似 IoU 但對小區域更敏感

**輔助指標**：
- Pixel accuracy（會被背景灌水，僅參考）
- Confusion matrix（pixel-level）
- 影像層級：每張圖預測「主瑕疵類」是否正確（業務常用）

**禁用 pixel accuracy 單一指標判定**。

## 7. 風險與備案

| 風險 | 觸發條件 | 備案 |
|---|---|---|
| 小類別 IoU < 0.2 | M6 後仍未改善 | 合併相近類 / WeightedSampler / 補拍標 |
| Batch effect 主導 | Train mIoU 高、val 爛 | 強化 ColorJitter + Gamma；考慮灰階輸入 |
| 背景灌水蓋過瑕疵 | 模型全預測背景仍 acc 高 | 換指標看 mIoU、加 Dice loss |
| EfficientNet-B0 容量不足 | Train mIoU 也卡住 | 換 efficientnet-b3 或 ConvNeXt encoder |
| 過擬合 | Train-val gap 大 | 強化 augmentation、降 epochs、dropout |
| 1201→512 resize 丟細節 | 小瑕疵 IoU 特別低 | 改 768×768 或 padding 不縮放 |

## 10. 起手式程式碼

### 模型定義（3 行）
```python
import segmentation_models_pytorch as smp

model = smp.Unet(
    encoder_name="efficientnet-b0",
    encoder_weights="imagenet",
    in_channels=3,
    classes=13,  # 12 瑕疵 + 1 背景
)
```

### Loss
```python
ce_loss = nn.CrossEntropyLoss(weight=class_weights)
dice_loss = smp.losses.DiceLoss(mode='multiclass')
def criterion(pred, mask):
    return ce_loss(pred, mask) + dice_loss(pred, mask)
```

### Metric
```python
from torchmetrics import JaccardIndex
miou = JaccardIndex(task='multiclass', num_classes=13, average='macro')
```

## 11. 跟原計畫的關係

- 原本走 **EfficientNet 純分類**：因為以為只有 image-level 標註
- 發現有 polygon GT → 升級到 **U-Net + EfficientNet encoder semantic segmentation**
- Encoder 仍是 EfficientNet，研究脈絡延續
- 跟 YOLO 專案形成方法論對照：
  - YOLO 專案 = instance detection/seg、anchor-based、速度導向
  - 本專案 = semantic segmentation、encoder-decoder、邊界精細導向

## 12. 後續延伸

- 換 decoder 比較：U-Net vs U-Net++ vs DeepLabV3+ vs FPN（smp 都支援）
- 換 encoder 比較：EfficientNet-B0/B3 vs ResNet50 vs ConvNeXt-Tiny
- TTA（test-time augmentation）提升最終 mIoU
- 模型壓縮 / ONNX 匯出
- Active learning：用模型挑信心低的樣本補標
